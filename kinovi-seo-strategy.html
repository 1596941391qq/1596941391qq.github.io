Dual-Domain Audit

kinovi.ai 不是技术底盘错了，而是把正确的双域架构，做成了一个内容和转化都太薄的站。

这版结论只做一件事：把上下文重新对齐。前提是 kinovi.ai + seedance2-pro.com 的双域收敛策略成立，所以第三条线不再提出“把 canonical 修回 kinovi.ai”这种自我冲突动作。真正的问题是页面体量、说服链路、商业解释和信任信号没有跟上。

Executive Thesis

核心判断

kinovi 的机会点是真实存在的。Seedance 2.0 的音频能力、角色一致性和视频生成卖点，足够支撑一套有竞争力的 AI 视频品牌叙事。问题不是“有没有差异点”，而是站内几乎没有把这些差异点变成可索引、可比较、可引用、可转化的页面资产。

现在做对了什么

双域架构本身合理，canonical、robots、sitemap、308 跳转逻辑总体上是一个“收口 SEO 域 + 展示品牌域”的成熟方案。这个前提不能在执行层被自己推翻。

现在最伤的是什么

全站只有 6 个可讲故事的页面，Gallery 这个最像证据的页面还被 noindex 处理，Pricing 的 Buy Now 直接断链，Blog 不成体系，导致搜索、AI 引用和注册转化都接不上。

Priority Issues

最该先处理的 4 个问题

P0转化断裂

Pricing 页的 Buy Now 是空链接

这不是小 bug，而是商业漏斗最后一段直接掉到底。你可以接受文章没写完，但不能接受用户走到付款前一步时什么都点不开。

建议：先修付款跳转，再补“套餐差异 / 免费额度 / 退款与信用点规则 / 常见顾虑 FAQ”四件套。任何内容增长都不该先于这一步。
P1证据被藏

Gallery 是最像销售证据的页面，却被 noindex + 无 H1 + 无专属 Schema

对 AI 视频产品来说，Gallery 不只是“作品集”，而是最接近真实结果页、最容易被比较、最适合承接长尾场景词的资产。把它关掉，相当于主动放弃了最能说服搜索引擎和用户的那一页。

建议：确认 noindex 是否为故意策略。如果不是，就把 Gallery 升级成“可索引的场景 / 风格 / 模板 / 成片案例中心”，并为单条作品或聚合分类补充可被引用的文本说明。
P1主题信号混乱

Homepage 双 H1 把主叙事稀释了

在“AI video generator”这种竞争极强的词上，首页不是不能讲多个卖点，而是不能在结构上像两个首页在抢话筒。当前更像视觉模块优先，而不是搜索语义优先。

建议：保留一个主 H1 说清“kinovi 是什么”，次级卖点全部下放到 H2 和摘要模块，不要让标题系统自己互相竞争。
确认项策略别乱改

双域策略本身成立，第三条线只能做校验和补强，不能反手推翻

如果既认定 seedance2-pro.com 是 SEO 收口域，又在执行路线里写“全站 canonical 修回 kinovi.ai”，那整份方案就逻辑自毁。第三条线的正确动作是确认双域都在 GSC、sitemap 提交完整、跳转覆盖所有路径、社交卡片与 Schema 指向规则一致。

这次必须锁死一个前提：第三条线不是改策略，而是验证策略。

Voice Direction

语气感与页面气质建议

品牌语气

像实验室里的创意工具，不像廉价 AI 拼贴站

文案基调应该是 “calm confidence + maker credibility”。少喊“best”与“magic”，多写“结果差异、参数、时长、成片效果、适合谁”。

页面写法

先给判断，再给证据，再给下一步

每个核心页面首屏都应有 TL;DR、结果表格、适用人群、限制条件和 CTA。不是堆功能，而是先帮用户判断“这是不是我要的工具”。

视觉风格

Claude / Insights 方向：克制、编辑感、信息密度高

避免廉价 SaaS 模板的彩色卡片堆砌。用温白底、窄色域、留白、大字标题、精细边框、审计式表格，做出“研究简报”而不是“营销海报”的气质。

建议定调句：kinovi 不该把自己包装成“什么都能生成”的全能 AI，而该包装成“对视频结果更敏感的创作者工具”。
Four-Line Strategy

四线方案

Line 1

导流页与比较页：去抢“正在做决策的人”

目标：吃下竞品比较 / 替代 / 限制 / 问题词

第一条线不需要重新定义双域，而是要围绕 seedance 相关热词、竞品对比和用户问题，生产一批高意图页面。重点不是“大而全”，而是每篇都能把一个决策问题讲透。

优先词组
  • `runway vs seedance / kinovi`
  • `sora alternative no waitlist`
  • `kling vs seedance 2.0`
  • `best ai video generator with audio`
页面模板
  • 首段先回答谁更适合谁
  • 中段做参数 / 价格 / 时长 / 音频 / 一致性表格
  • 结尾导到主站功能页或试用入口
不能犯的错
  • 只写“比竞品更强”但没有证据
  • 把导流页写成泛教程
  • 导流页和主站之间没有明确内链分工
Line 2

主站内容系统：把“6 页壳子”升级成可累积权威的站

目标：建立 topical authority 与产品说服体系

第二条线的任务不是“开 blog 栏目”，而是建立 4 种页面资产：支柱页、场景页、比较页、实验页。客户会看到的应该是内容结构、页面职责和商业价值，而不是内部生产规格。

内容结构
  • 支柱页负责定义核心能力与行业判断
  • 比较页负责替用户缩短决策时间
  • 场景页负责把功能翻译成真实使用场景
  • 实验复盘页负责沉淀证据、结果和可引用素材
优先主题
  • AI video with native audio
  • Character consistency benchmarks
  • Text-to-video workflows for marketers
  • Seedance 2.0 vs mainstream tools
页面结构
  • TL;DR
  • 参数/结果表
  • 谁适合 / 谁不适合
  • FAQ + CTA + 相关内链
Line 3

技术 SEO：验证并加固双域，而不是推翻双域

目标：让正确策略稳定执行

第三条线只做两件事：一是把现有双域策略校验完整，二是把最影响索引和理解的硬伤补齐。这里绝对不再出现“canonical 改回 kinovi.ai”这种反向动作。

确认项
  • kinovi.ai 与 seedance2-pro.com 均完成 GSC 验证
  • sitemap 已在双域层提交
  • 308 覆盖所有应跳转路径
  • og:url / schema url 规则与策略一致
硬伤修复
  • 修首页双 H1
  • 补 Gallery H1 与文本说明
  • 确认 Gallery noindex 是否应移除
  • 修 Pricing 的购买链路
AI Search 补强
  • 核心页首段直接回答问题
  • 更多表格而不是空泛段落
  • FAQ 覆盖价格、限制、时长、音频、商用
  • 案例页补结构化数据
Line 4

外链与品牌证据：让“可被引用”变成站外事实

目标：推排名，也推信任

第四条线不应只理解成传统外链建设。对 kinovi 这种 AI 工具站，更值钱的是“可引用的比较资产、实验资产、目录资产和真实用户反馈资产”。

高价值资产
  • AI video benchmark 报告
  • Prompt 模板包与参数表
  • 竞品价格 / 效果比较表
  • 真实案例前后对比页
优先渠道
  • AI 创作者博客与通讯
  • 开发者内容平台
  • 产品目录 / review 平台
  • 创作者社区 roundups
站内配合
  • 收集用户评价并上墙
  • 展示社区规模与作品数据
  • 把第三方 mention 做成 trust strip
  • 每个核心页都有“证据模块”
Execution Roadmap

执行路线

第 1 周

止血

修 Buy Now、修首页 H1、确认 Gallery noindex、完成双域 GSC 与 sitemap 校验。这里的目标不是增长，是先把漏斗和策略前提修正到不自相矛盾。

第 2-3 周

补最少但最关键的内容资产

上线 3 篇比较页、2 篇场景页、1 篇支柱页、1 个可索引 Gallery 聚合页。先做能解释“为什么选 kinovi”的资产，不要先铺泛流量教程。

第 4-8 周

建立系统

把第二条线扩到 12-18 个核心页面，把第四条线的实验资产和第三方 mention 一起推起来，同时重做主站 trust 区和 FAQ 区。

第 9-12 周

放大

根据排名与转化数据，继续扩比较矩阵、场景页与实验复盘页，形成“导流页截流 + 主站收口 + 双域稳定 + 站外证据增信”的闭环。

这版方案的关键修正只有一句话：先承认双域策略，再围绕它做四线闭环。 如果前提不稳，后面的每一条执行建议都会互相打架。
