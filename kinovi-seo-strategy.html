SEO Audit / Four-Line Strategy

kinovi.ai SEO 审计与四线策略

这不是一份轻量体检。它先把站点真正卡住增长的地方拆开，再给出一套能把搜索流量、AI 引用、信任建立和注册转化接上的执行路线。

核心判断

kinovi.ai 现在的问题，不是技术底盘完全没有，而是能拿去竞争的页面资产还没长起来。产品卖点已经能打，但站内表达、证据沉淀、比较页矩阵、案例密度、站外引用和转化收口都还不够，导致搜索系统能看到它，却还不愿意把更高价值的流量持续交给它。

一句话结论

底盘能用，增长资产太薄，收口动作太弱，第四条线几乎还没真正启动。接下来要做的不是零碎修补，而是把页面、证据、内链、外链和转化模块一起拉起来。

Kinovi 已经有值得写进搜索结果的产品差异点，现在缺的是把这些差异点做成别人看得懂、愿意点、愿意引、愿意注册的页面。
Executive Summary

执行摘要

这站已经过了“从零开始”的阶段，但还没到“可以靠内容系统稳定吃增长”的阶段。现状更像一个有产品、有卖点、有一定技术秩序的主站壳子，离真正的 SEO 资产库、GEO 引用源和转化中枢还差一层厚度。

已经具备的基础

双域分工逻辑基本成立，canonical、robots、sitemap、跳转方向没有彻底失控，页面也不是完全没有语义结构。换句话说，这不是一个要先救火的废站，而是一个可以直接进入增长建设阶段的半成品。

真正拖后腿的地方

比较页和导流页矩阵还没长出来，Blog 没有形成支柱页与集群页，Gallery 这种天然能吃证据流量的资产没有被当成索引层重点经营，Pricing 的收口动作也不够硬。内容、信任、转化和站外放大没有形成连锁反应。

Priority Findings

关键问题

P0收口断裂

Pricing 与交易路径没有把高意图用户稳稳接住

用户已经走到最靠近注册和付费的一步时，页面的说服力、风险解释和 CTA 设计还不够强。价格不是唯一问题，真正的问题是页面没有把“为什么现在就该试”和“为什么它比替代品更值得试”说透。

P0资产太薄

站点还没有长出可以稳定吃词的页面规模

只有少量核心页，平均内容厚度约 676 词，无法支撑高竞争 AI 视频赛道的持续增长。这个问题不是靠补几篇 blog 能解，必须做成比较页、支柱页、案例页、实验页和资源页一起扩的体系工程。

P1证据缺位

产品卖点有，但搜索结果里缺“可验证的理由”

如果用户看到的只是功能描述，他会继续搜索第二篇、第三篇。Kinovi 需要把速度、音频、稳定性、风格一致性、成本和使用边界做成对比表、案例、实验和常见问题，才能把点击和停留真正留住。

P1第四线偏空

站外 SEO 还没有形成系统动作

目前看不到明确的可链接资产路线，也没有把竞品链接缺口、品牌提及、资源页链接、目录、播客、社区壳层和 PR 线串起来。第四条线如果继续空着，前面三条线的收益会被放大不足。

P1重点页策略不清

Gallery、Docs、Case、Comparison 各自该扮演什么角色，还没有完全写进站点结构

这类 SaaS 站最怕所有页面都在讲同一种话。现在的站点已经显出这个倾向。后面必须给每类页面一个明确角色，不然内容会越来越多，但站点不会越来越强。

Line 1

第一条线：导流页与比较页矩阵

Line 1

先吃最接近决策的一层搜索意图

替代词 / 对比词 / 最佳工具词 / 问题词

第一条线不是普通 blog。它更像一个会做判断的编辑部，专门去截用户正在犹豫、正在比较、正在挑替代品的那一刻。用户搜的是 Runway、Sora、Kling、Pika、Luma，脑子里想的不是品牌故事，而是结果、速度、价格、音频、稳定性、等待时间和可控性。谁把这些点说透，谁就更容易拿走点击。

Kinovi 这条线最该打的不是“我们也可以做视频”，而是“你为什么在这一类场景里应该选我们”。比较页如果没有参数、对比表、边界说明、样例结果、价格解释和适合谁不适合谁，它就只是一篇把竞品名称写上去的空壳。导流页真正值钱的地方，在于它帮用户把决策过程缩短了一半。

优先词包

先抓高意图词：`runway alternative`、`sora alternative`、`kling alternative`、`seedance 2.0 vs runway`、`best ai video generator with audio`、`ai video generator for ads`、`kinovi review`。

先做会带来注册的人，再做会带来流量的人。顺序不能反。

页面模板

每篇比较页都应该有：结论摘要、对比表、适合人群、失败场景、价格与额度、样例结果、常见疑虑、CTA。这样页面才能接住“已经快下单”的人。

执行节奏

第一批先做 5 篇核心比较页和 3 篇场景页。比较页带注册 CTA，场景页带模板和案例 CTA。两者一起跑，避免只有流量没有收口。

建议优先做的页面族
页面族要解决的问题
替代页直接吃竞品不满流量，把“为什么换”说清楚。
对比页把价格、音频、生成质量、风格一致性、速度放到一张表里。
Best-of 页吃工具型商业词，用编辑视角拉到自己的比较体系里。
场景页回答“适不适合我”，例如广告、电商、社媒、短视频脚本。
问题页回答等待时间、音频质量、角色一致性、延展能力这类具体疑虑。
这一条线最怕什么

最怕写成泛泛的工具博客。没有真实判断，没有主张，没有表格，没有边界，就算收录了也很难把用户留下来。更糟的是，这类页还会稀释全站气质，让品牌显得像一个在追关键词的普通工具站。

Kinovi 应该用“实验感”和“编辑判断”去写这一层。写得像真的用过、真的测过、真的知道差异点，而不是把竞品参数抄一遍。

Line 2

第二条线：主站内容系统与品牌权威

Line 2

把主站做成行业问题入口，而不是只有产品入口

Pillar / Cluster / Cases / Resources / Docs

第二条线负责把站点从“介绍产品”拉到“解释行业问题”。用户不应该只在站里看到功能清单，还应该看到方法、限制、案例、对比、模板、流程、FAQ、术语解释、操作建议和成本判断。只有这样，主站才会慢慢长出权威感。

这条线也决定后面第一条线能不能接得住。比较页截来的人，如果点进主站之后只看到一个泛泛的首页和一页不够能打的定价页，流量还是会漏。主站必须有能力继续把人往深处带：从比较走到解释，从解释走到案例，从案例走到模板和注册。

主站应该优先长出来的内容角色
  • 支柱页：解释 AI 视频生成的大问题，拿行业词。
  • 集群页：拆具体问题，吃长尾词和 People Also Ask。
  • 案例页：给出真实输入、输出、限制和结果。
  • 实验页：用同一组 prompt 和设置跑多工具横评。
  • 资源页：模板包、脚本包、Prompt 包、成本表、选型表。
  • Docs / FAQ：承接开发者与高意图用户的细问题。
建议先做的支柱主题
  • AI Video Generation 全景指南
  • Text-to-Video with Audio 实战指南
  • Character Consistency Benchmark
  • AI Video Cost and Speed Guide
  • AI Video for Ads / E-commerce / Social Media
  • Seedance 2.0 工作流与最佳实践
第二条线的写法要求

这里不能写成“品牌宣传文”。真正能吃词的支柱页，一定会解释做法、限制、失败原因、替代方案和适用人群。也就是说，它不是单向夸品牌，而是先让用户觉得“这页真的有用”，再自然把品牌放进解决方案里。

第二条线该长什么样

当这条线做顺之后，主站会开始像一个活的资料库。用户搜行业问题能进来，搜教程能进来，搜案例能进来，搜选型和成本也能进来。站点不再只有品牌词入口，而是有一整圈问题入口。

Line 3

第三条线：技术 SEO 与 AI Search Ready

Line 3

把双域底盘校准到稳定、可解释、可引用

抓取 / 索引 / Schema / 性能 / AI 引用

第三条线不负责推翻前提，它负责把前提做实。既然这次已经明确是双域策略，那技术线就应该围绕这个前提做校验、治理和放大：Google 抓到的是不是对的版本，重点页能不能稳定收录，Schema 是否在讲同一件事，页面是不是足够可摘取，站内是不是存在自我打架的语义结构。

这条线最怕两件事。第一件事是执行层自己把前提打乱，前面承认双域成立，后面又条件反射式地把 canonical 改回主域。第二件事是技术正确但业务无感，分数看起来漂亮，结果最值钱的页面还是没词、没点、没注册。第三条线的工作必须服务增长，而不是只服务审计表。

先做的技术治理

核对双域 GSC、sitemap 提交、canonical 指向、308 跳转、重点页索引状态、首页 H1、Gallery 策略和重点页 Schema 组合。

AI Search Ready 的重点

让首页、功能页、比较页、案例页、支柱页都具备 TL;DR、定义块、步骤块、对比表、FAQ、限制说明和结论块。这样更容易被 AI 系统摘取。

技术线的真正目标

不是把分数刷到好看，而是让高价值页能被看见、被理解、被引用、被点击，并顺利把用户送到注册动作上。

建议补齐的 Schema 组合
页面类型建议组合
首页Organization + WebSite + SoftwareApplication
功能页SoftwareApplication + FAQPage + BreadcrumbList
比较页Article + FAQPage + BreadcrumbList
案例页Article + VideoObject
教程页HowTo + FAQPage
Gallery 聚合页CollectionPage + BreadcrumbList
这一条线的重点检查口径

看的是重点页是否能稳定承接增长，而不是把全站每一个小问题都当成同等优先级。首页、Pricing、比较页、功能页、Gallery、案例页，是技术线的核心检查对象。

技术线是增长线的地基，不是独立展示柜。

Line 4

第四条线：站外 SEO、证据资产与品牌壳层

Line 4

把站内的判断和实验，变成别人愿意链接和转述的材料

Backlinks / Mentions / PR / Communities / Shells

第四条线不是“发点外链”这么简单。它做的是另一件更值钱的事：把主站上的比较、案例、模板、实验、成本表和行业判断，推出去变成行业里别人愿意引用的证据源。站内如果做对了，第四条线会把前三条线放大；站内如果做得太薄，第四条线就会变成一堆没有锚点的零散动作。

Kinovi 这条线最适合打实料。不要空喊品牌口号，要拿得出可以被引用的东西。比如：同一组 prompt 下 6 个工具的生成结果；30 秒与 60 秒视频的成本、失败率和等待时间；有无原生音频时的体验差异；角色一致性在多镜头叙事下的稳定性；广告、电商、社媒三个场景的成片表现。外部站点愿意链接的，通常不是广告，而是这些别人懒得自己做的整理和实验。

第一组动作：可链接资产
  • AI 视频基准测试：同 prompt、同参数、同评价维度。
  • AI Video Cost Index：不同工具的价格、额度、失败成本。
  • Prompt 模板包：广告、社媒、电商、分镜四类模板。
  • 选型表：适合谁、不适合谁、什么时候别选它。
  • 案例库：每个案例都带输入、参数、输出、失败点。
第二组动作：链接获取打法
  • 竞品反链截流：拆竞品比较页、资源页、工具页的链接来源。
  • Broken Link Building：找失效的 AI 工具榜单、教程、资源页。
  • Unlinked Mentions：回收品牌提及但未加链接的帖子、文章、Newsletter。
  • Testimonial Links：给上下游工具写真实反馈换回品牌页链接。
  • Resource Page Outreach：把模板包、基准测试、成本指数投给资源页。
第三组动作：品牌壳层与可控外部阵地

第四条线不能只赌别人给不给链接，也要建立自己的外部壳层。包括目录站、工具导航、创作者社区、GitHub 资源页、Medium / Dev.to / Hashnode 风格的壳层、Reddit 与 Quora 的高质量解释帖、Newsletter 投稿、播客访谈页、创作者案例采访页。这些阵地不只是为了出链接，更是为了让品牌词 SERP 更立体。

第四组动作：更实用的奇招

这条线真正好用的“奇技淫巧”并不花哨，核心是借别人现成的流量池放大自己的实验资产。比如把一份 AI 视频基准表拆成三种版本：完整长文放主站，精简表投给通讯和工具目录，可嵌入版本做成图片或 Notion 镜像，方便别人直接引用。再比如把“某竞品更新后变慢或涨价”的时间窗口做成快反页，专门截流一波不满搜索。

优先目标页

外链不要全打首页。优先打比较页、支柱页、实验页、资源页、案例页。首页只吃品牌信任型链接，增长链接要投给有词、有表格、有结论的页面。

优先渠道

AI 工具榜单、创作者 Newsletter、开发者博客、行业媒体、视频创作社区、设计与营销社区、工具导航、资源页、播客 show notes、GitHub 工具集合。

这条线做顺之后

品牌词 SERP 会更厚，站内强页会得到更多支持，AI 引用更容易发生，主站也会从“某个工具”慢慢变成“这类工具里那个认真做对比和实验的人”。

Growth & Conversion
