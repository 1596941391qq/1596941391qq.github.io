Executive Summary

Critical Issues

关键问题清单

P0直接损失

Pricing 页的 Buy Now 是空链接

这里掉的不是体验分，是现金流。用户愿意走到付款前一步，说明说服成本已经花掉了，最后点不开，前面所有流量成本都白费。

先修支付入口，再补套餐解释、免费额度、信用点规则、退款说明和 FAQ，页面才有资格承接商业词。
P1证据资产

Gallery 页面被 noindex，还是无 H1、无主力 Schema 的状态

AI 视频工具最值钱的页面类型之一就是成片展示页。它离“真实结果”最近，离“比较判断”最近，也最适合承接风格词、场景词、质量词和实验词。

Gallery 这块一旦恢复索引，再加分类文本、成片说明、单项作品页、FAQ 和 VideoObject，站点气质会立刻从“工具壳”向“可验证产品”靠近。
P1语义结构

Homepage 双 H1 把首页主命题拉散了

首页应该负责一件事：告诉用户和搜索引擎，kinovi 是什么、有什么差异、下一步去哪里。两个 H1 会让这个命题失去重心。

保留一个主 H1，其他卖点下放到 H2、摘要卡、比较表和证据模块，首页阅读体验会清楚很多。

Line 1

第一条线：导流页与比较页矩阵

Line 1

先抓住已经在做决策的人

竞品截流 / 替代词 / 比较词 / 问题词

这一条线负责吃下最接近注册和付费的一层搜索意图。用户搜 Runway、Sora、Kling、Pika、Luma、Hailuo 的时候，脑子里想的不是“我要不要了解 kinovi”，而是“我现在手里的选择够不够好，贵不贵，稳不稳，能不能出片”。谁能在这个时刻把问题说透，谁就有资格拿走点击。

Kinovi 这一层不能写成泛泛的 AI 工具博客。它要更像一个带着测试偏见的编辑部：知道主流产品各自卡在哪，知道哪些词背后是真付费意图，知道一篇比较页如果没有参数、没有结果、没有适用边界，读者读完还是会回去继续搜第二篇。导流页真正值钱的地方，不在于把竞品名字挂上去，而在于把决策过程截断。

优先关键词方向
  • `runway alternative`
  • `sora alternative no waitlist`
  • `seedance 2.0 vs runway`
  • `best ai video generator with audio`
  • `kling vs seedance`
页面类型
  • 竞品对比页
  • 替代方案页
  • 限制 / 故障排查页
  • 价格对比页
  • 场景应用页
页面结构
  • 首段直接给判断
  • 中段给参数与结果表
  • 尾部给适用人群与 CTA
  • FAQ 覆盖价格、质量、限制、音频、商用
应该先打的内容簇
  • 竞品替代簇：谁贵、谁慢、谁限制多，kinovi 在哪里更顺手
  • 竞品对比簇：Seedance 2.0 与主流视频模型的正面硬碰硬
  • 功能问题簇：音频、字幕、时长、分辨率、渲染时间、导出限制
  • 场景决策簇：广告团队、短视频团队、独立创作者、电商卖家
这条线最容易写废的地方
  • 写成“十大最佳 AI 工具”那种谁都能复制的水文
  • 只说功能，不说用起来到底哪里卡、哪里省时间
  • 把 kinovi 强塞进结论，反而显得心虚
  • 没有表格，没有结果，没有一句能被用户带走的判断

导流页该有点锋利。该说某个竞品贵，就直接说它贵。该说某个模型画面强但音频弱，就直接说它音频弱。读者不是来听客套话的，他是来缩短决策时间的。每篇文章只要真帮他省下十分钟，品牌就往前走一步。

这一条线的产出也不能只盯着 Google。写得好的比较页，天然适合被 X 线程、Reddit 讨论、创作者通讯、邮件摘要二次利用。Google 拿自然流量，社媒拿情绪扩散，主站拿最后的注册收口。导流页不是旁支，它是矛头。

Line 2

第二条线：主站内容系统与品牌权威

Line 2

把主站做成行业问题入口，而不是只有产品入口

Pillar / Cluster / Cases / Experiments

主站承接的是长期价值。这里需要建立一整套能解释产品、解释场景、解释限制、解释结果的内容系统。用户来到主站之后，应该能看到“它是什么、适合谁、怎么用、和别家比怎样、能做出什么、值不值得买”的完整答案链。现在这条链缺口很大，所以站内看起来更像功能说明，离真正能把行业问题接住的品牌还有距离。

Kinovi 主站眼下最该补的是内容角色。哪些页面负责解释行业，哪些页面负责收口比较意图，哪些页面负责给开发者一个明确入口，哪些页面负责拿下 AI 引用，哪些页面负责把半信半疑的人变成注册用户。这些角色如果不分清，后面就会出现一种很典型的无效勤奋：文章一直在发，站点还是没有权威感。

建议先做的支柱主题
  • AI Video Generation 2026 全景指南
  • Text-to-Video with Audio 实战指南
  • Character Consistency Benchmark
  • AI Video for Marketers / Creators / Filmmakers
  • AI Video API 指南与开发文档桥接页
建议扩的集群方向
  • 竞品对比与替代方案
  • 场景页：广告、电商、教育、短视频、游戏、播客
  • 问题页：渲染时长、音频、字幕、分辨率、导出
  • 实验页：盲测、成本、时长、质量复盘
  • 资源页：Prompt 模板、对比表、下载包
主站应该长出的几个固定栏目
  • Comparison Lab：把所有 vs / alternative / pricing judgement 集中起来
  • Use Case Library：广告、电商、课程、播客、YouTube、游戏宣传片
  • Experiment Log：盲测、成本测试、时长测试、失败案例复盘
  • Resource Hub：模板、参数表、Prompt 包、脚本包、对比表下载
这一条线真正该追的效果
  • 让用户不需要回 Google 二次确认
  • 让 AI 搜索系统能稳定抽到结构化答案
  • 让内容页自己给产品页输送判断型流量
  • 让品牌从“工具名”慢慢长成“某类问题的默认答案”
内容组织方式
内容层级职责应承接的搜索意图
支柱页定义能力、建立权威、做全景导航品类词、主题词、行业词
比较页压缩决策时间、承接商业词vs / alternative / pricing / best
场景页把功能翻译成使用价值for marketers / for YouTube / for ads 等
实验页沉淀差异证据与外链素材benchmark / quality / cost / test
资源页承接下载和留资template / checklist / comparison sheet

这一条线写得好的时候，站内会开始出现一种很稳的感觉：同一个主题，不是靠一篇文章硬顶，而是支柱页、比较页、案例页、FAQ 页和下载页一起发力。Google 会更容易理解站点主题，用户会更容易感受到“这个品牌确实懂我在问什么”，转化也不会只压在首页和 pricing 页身上。

写得差的时候也很明显。你会看到一堆标题看起来都对，内里却谁也不带谁，文章之间没有角色分工，最后每篇都像一个孤岛。这种内容体系短期能凑页面数，长期很难凑出权威感。Kinovi 现阶段要避开的，就是这种虚胖。

Line 3

第三条线：技术 SEO 与 AI Search Ready

Line 3

把已有双域底盘校准到稳定、可解释、可引用

抓取 / 索引 / Schema / 性能 / AI 引用

第三条线负责消化策略前提并保证执行质量。这里不再讨论是否切回单域，动作集中在双域校验、索引治理、结构化数据、性能和可引用表达上。搜索引擎与 AI 引用系统看到的，应该是一套有层级、有答案、有证据的页面结构，而不是零散的功能碎片。

技术线在这种项目里最怕两件事。第一件事是自我打架，前面刚承认双域是收口方案，后面又像条件反射一样把 canonical 往主域改。第二件事是技术正确、业务无感，修了一堆分数，结果最值钱的页面还是没收录，最该转化的按钮还是点不开，最该解释的卖点还是没人看懂。技术线要做的是把增长路径打通，不是为了交一张工整的审计表。

抓取与索引
  • 核对 robots.txt 与 sitemap 覆盖
  • 确认双域 Search Console 数据
  • 确认 Gallery 与重点页的索引策略
  • 检查 4xx / 5xx / 跳转链 / 参数页
结构化数据
  • 首页：Organization + WebSite + SoftwareApplication
  • 功能页：FAQPage + BreadcrumbList
  • 案例页：Article + VideoObject
  • Gallery：分类页与作品页双层结构
AI Search 表达
  • 首段给 TL;DR
  • 中部给结果表格
  • FAQ 说清限制与适用边界
  • 案例页写清参数、结果和结论
技术线的真 KPI
  • 高价值页被稳定抓取，而不是参数页被乱抓
  • Gallery、比较页、案例页都能清楚表达页面主命题
  • Schema 不只是“有”，而是落在对的页面、对的层级
  • AI 搜索抽取到的是完整答案，不是残缺句子
最值得优先盯的风险
  • 双域 Search Console 数据不完整，导致判断失真
  • Gallery 如果长期 noindex，会持续浪费最强资产
  • 首页和 pricing 页如果不补结构，商业词很难稳定提升
  • 性能如果拖垮首屏，比较页和场景页的阅读深度会先掉

技术线处理得好，Kinovi 的页面会开始像“可以直接摘录答案的知识单元”。处理得不好，整个站会一直停留在“能打开、能演示、能注册，但讲不清自己”的阶段。对 AI 工具站来说，这个差距非常致命，因为越是新产品，越需要页面帮它说服人，而不是把理解成本留给用户自己消化。

Line 4

第四条线：外链建设、品牌证据与站外放大

Line 4

让主站的判断、表格和实验在站外也被看见

Backlinks / Mentions / Assets / PR

第四条线的核心工作，是让站内的优质资产往站外扩散，获得链接、提及、评价和二次引用。对 kinovi 这类工具站来说，站外最值钱的是比较资产、实验资产、模板资产和真实使用反馈。真正能带来持续收益的外链，不会只停留在“品牌被提到了一次”，它会把主站页面变成一个被别的创作者反复拿来引用的标准参考。

Kinovi 这一层最适合打的是实料，不是漂亮话。单纯说“我们是一家很棒的 AI 公司”，没人会愿意给链接。更有效的说法是：我们把 6 个工具放在同一组 prompt 和参数下测了，结果差异在这里；我们把 30 秒和 60 秒的成本、质量、失败率摊在表里；我们把字幕、音频、一致性、导出这些用户真的会卡住的细节做成可下载模板。站外要的是可引用材料，广告词没有人转述。

第四条线该长什么样
  • 品牌词搜索页里，出现更多可控或可影响的壳层
  • 对比页和实验页持续拿到相关站点引用
  • 第三方平台开始为主站补信任信号
  • 创作者会主动拿 kinovi 的实验图表做二次传播
这条线最怕什么

  • 目录站上了，但主站本身没准备好承接信任
  • 外链都打到首页，比较页和资源页拿不到支持
  • 品牌词外面很热闹，站内却没有一个能闭环的强页

外链这件事，真正拉开差距的从来不是发了多少封邮件，而是对方为什么愿意给你这条链接。Kinovi 如果手里只有普通产品介绍页，能换来的大多是普通结果。Kinovi 如果手里有别人做不出来、懒得做、又确实有用的比较表和实验页，整个外链获取难度会直接降一个档位。

第四条线做顺之后，主站的气质会变。品牌不再只是“某个 AI 视频工具”，而会更像“这类产品里有人专门在认真做测试、做比较、做模板、做解释的那个玩家”。这层形象一旦长出来，SEO、GEO、转化和口碑会开始互相抬轿子。
